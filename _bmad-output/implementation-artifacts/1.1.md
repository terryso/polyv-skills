### 插件配置规范 [Source: architecture.md]

**`.claude-plugin/marketplace.json`:**
```json
{
  "name": "polyv-skills",
  "owner": {
    "name": "Nick"
  },
  "metadata": {
    "description": "polyv直播平台 API Skills 集合"
  },
  "plugins": [
    {
      "name": "polyv-skills",
      "source": "./",
      "description": "polyv直播平台 API 工具包 - 创建频道、管理等",
      "homepage": "https://github.com/terryso/polyv-skills",
      "license": "MIT",
      "keywords": ["polyv", "live-streaming", "api", "skills"]
    }
  ]
}
```

**`.claude-plugin/plugin.json`:**
```json
{
  "name": "polyv-skills",
  "version": "1.0.0",
  "description": "polyv直播平台 API Skills 集合",
  "skills": ["./skills/"],
  "keywords": ["polyv", "live-streaming", "api"]
}
```

### 安装方式支持 [Source: architecture.md]

此结构支持以下安装方式：
1. **Claude Code 插件市场**（推荐）
   ```bash
   /plugin marketplace add terryso/polyv-skills
   /plugin install polyv-skills
   ```
2. **npx skills**
   ```bash
   npx skills add terryso/polyv-skills
   ```
3. **Git Submodule**
   ```bash
   git submodule add https://github.com/terryso/polyv-skills.git .agents/polyv-skills
   ```
4. **Clone & Copy**
   ```bash
   git clone https://github.com/terryso/polyv-skills.git
   cp -r polyv-skills/skills/* .agents/skills/
   ```

### .gitignore 规范

应包含：

- 配置文件中的敏感信息（`~/.polyv-skills/config.json` 不提交）
- IDE 配置文件
- macOS/Windows 系统文件

### Project Structure Notes

**目录用途说明:**
- `.claude-plugin/` - Claude Code 插件配置，用于插件市场安装
- `skills/` - Skill 定义文件，每个 Skill 一个子目录
- `tools/clis/` - CLI 工具脚本（纯 JavaScript）
- `tools/integrations/` - 集成参考文档
- `tools/REGISTRY.md` - 工具注册表，列出所有可用工具
- `config/` - 配置模板

**命名规范 [Source: architecture.md]:**
- Skill 目录: `polyv-{action}-{resource}` 格式
- Skill 文件: 固定名称 `SKILL.md`
- CLI 文件: `kebab-case.js` 格式
